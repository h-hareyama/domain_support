# HP公開 ドメイン判定ツール

> プロモ事業のHP公開時に、園の状況に応じた必要ヒアリング項目を自動で絞り込むツールです。

---

## 概要

園のHP公開・ドメイン管理は園ごとに状況がバラバラ（新規取得？移管？メールあり？旧サイトあり？）で、ディレクターが何を確認すればいいか判断が難しい状況でした。

このツールは、**ディレクターがYes/No・選択式で答えていくと、その園に必要なヒアリング項目だけが動的に表示される**仕組みです。

### 解決したい課題

- ディレクターが園に聞くべきことを迷わない
- 園の状況に応じて、必要な追加質問だけが出る
- テクニカルサポート側に判断に必要な情報が揃った状態で依頼が来る

---

## 使い方

### ディレクター向け

1. ツールのURLにアクセス（Slackで共有されたURL）
2. 5ステップに沿って入力
   - Step 01: 園の基本情報（園名・担当D・公開希望日）
   - Step 02: ドメインの扱い（新規／移管／他社継続）
   - Step 03: メールの扱い（なし／新規／既存継続）
   - Step 04: 旧サイトの有無 → ありの場合リダイレクト要否
   - Step 05: 結果シート（自動生成）
3. 結果シートを園に確認しながら埋める
4. 完了したら以下のいずれかで共有
   - **Markdownコピー** → Notion・Slackに貼り付け
   - **テキスト保存** → `.md`ファイルダウンロード
   - **印刷・PDF保存** → ブラウザの印刷機能
   - **Slack共有用文面** → 進捗サマリだけ短く貼り付け

### テクニカルサポート向け

ディレクターから共有されたMarkdownを受け取り、パターンIDとリスク評価を確認した上で作業に着手してください。

## パターンID の読み方

結果シートに表示される `パターンID` は4文字の組み合わせで構成されています。

```
DT-MC-OX
│ │  │ │
│ │  │ └─ リダイレクト: 0=不要 / S=自社対応 / X=他社依頼 / _=対象外
│ │  └─── 旧サイト: O=あり / _=なし
│ └────── メール: 0=なし / N=新規 / C=既存継続
└──────── ドメイン: N=新規 / T=移管 / X=他社継続
```

### 主要パターン例

| パターンID | 意味 | リスク |
| :--- | :--- | :--- |
| `DN-M0-__` | ドメイン新規・メールなし・旧サイトなし | 低 |
| `DN-MN-OX` | ドメイン新規・メール新規・旧サイトあり・他社依頼 | 中 |
| `DT-MC-OS` | ドメイン移管・メール継続・旧サイトあり・自社対応 | **高** |
| `DX-M0-OX` | 他社継続・メールなし・旧サイトあり・他社依頼 | 中 |

---

## リスク評価の基準

| リスク | 条件 | 注意点 |
| :--- | :--- | :--- |
| **高** | 既存メール継続を含むケース | MX/SPF設定ミスでメール停止のリスクあり |
| **中** | ドメイン移管 or 他社へのリダイレクト依頼を含む | リードタイムに余裕を持つ必要あり |
| **低** | 上記以外（シンプルケース） | 基本フローで問題なし |

## 質問項目の追加・修正方法

質問項目は `index.html` 内の `QUESTIONS` 配列で管理しています。

### 質問項目の構造

```javascript
{
  id: 'q-newdom-1',          // ユニークID（変更厳禁）
  cat: 'ドメイン',            // カテゴリ（基本/ドメイン/メール/旧サイト/リダイレクト）
  required: true,            // 必須項目かどうか
  q: '希望ドメインの候補（第1〜第3希望）',  // 質問文
  why: '取得できない場合の予備として',      // 確認理由
  cond: s => s.domain === 'new',           // 表示条件（state を受け取る関数）
  type: 'textarea'                          // 入力形式: text / textarea
}
```

### 表示条件の書き方

```javascript
// ドメインが新規取得の場合のみ表示
cond: s => s.domain === 'new'

// メール継続 かつ 旧サイトあり の場合のみ表示
cond: s => s.mail === 'continue' && s.oldsite === 'yes'

// 常に表示
cond: () => true
```

### state の取りうる値

```javascript
state = {
  domain: 'new' | 'transfer' | 'external',
  mail: 'none' | 'new' | 'continue',
  oldsite: 'yes' | 'no',
  redirect: 'none' | 'self' | 'external' | '',
}
```

## 既知の制限・注意点

- ヒアリング結果は**ブラウザ上のみで保持**されます。ページを閉じると消えるため、必ずMarkdown等で出力してください
- 個人情報の保存はしません（サーバ送信なし・ローカル完結）
- 質問項目の追加・削除は開発者（テクサポ）が実施します。要望があればSlack #tech-support チャンネルへ
