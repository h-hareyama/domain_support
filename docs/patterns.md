# パターン詳細

ヒアリング判定ツールが出力する全パターンの詳細です。

---

## パターンIDの構成

```
DX-MX-OR
│ │  │ │
│ │  │ └─ R: リダイレクト
│ │  └─── O: 旧サイトの有無
│ └────── M: メール
└──────── D: ドメイン
```

### 各文字の意味

| 軸 | 値 | 意味 |
| :--- | :--- | :--- |
| **D**（ドメイン） | `N` | 新規取得（お名前.comで新規） |
| | `T` | 移管（弊社管理へ） |
| | `X` | 他社管理継続（AレコードのみAWSへ） |
| **M**（メール） | `0` | なし |
| | `N` | 新規導入（GWS等） |
| | `C` | 既存メール継続 |
| **O**（旧サイト） | `O` | あり |
| | `_` | なし |
| **R**（リダイレクト） | `0` | 不要 |
| | `S` | 自社で対応 |
| | `X` | 他社へ依頼 |
| | `_` | 対象外（旧サイトなしの場合） |

## パターン別の主要対応フロー

### 🟢 低リスクパターン

#### `DN-M0-__` ：ドメイン新規・メールなし・旧サイトなし

**標準フロー：**
1. 園の希望ドメインヒアリング（第1〜第3希望）
2. お名前.comでドメイン取得
3. AWS Route 53でAレコード設定
4. studioと紐付けて公開

**所要期間目安：** 2〜3週間（`.ed.jp`の場合は実在証明で+1〜2週間）

---

#### `DN-M0-OS`：ドメイン新規・メールなし・自社リダイレクト

**標準フロー：**
1. 上記`DN-M0-__`に加えて
2. リダイレクトの種類確認（www有無 / 旧→新ドメイン）
3. 制作陣 or テクサポで設定
4. 公開日当日にstudio公開とリダイレクト同時実施

---

### 🟡 中リスクパターン

#### `DT-M0-O_`：ドメイン移管・メールなし

**標準フロー：**
1. 現管理会社からAuthCodeを園経由で取得
2. お名前.comで移管申請（3日〜10日かかる）
3. AWS Route 53でDNS設定
4. studioと紐付けて公開

**注意点：**
- ドメイン有効期限が残14〜30日切ってると移管不可
- 移管ロックがかかっている場合は解除依頼が必要
- AuthCodeは発行から7〜30日で失効

**所要期間目安：** 3〜4週間

---

#### `DN-MN-OX`：新規・メール新規・他社リダイレクト依頼

**標準フロー：**
1. 旧管理会社へリダイレクト事前依頼（公開3日前まで）
2. お名前.comでドメイン取得
3. GWS for Education 申込（メール用）
4. AWS Route 53でDNS設定（A + MX）
5. 公開日当日：studio公開・リダイレクト実施

**注意点：**
- 他社依頼は「公開日当日に合わせて実施」と明記
- 旧ドメインの契約継続を確認（切れるとリダイレクトも消える）

---

### 🔴 高リスクパターン

#### `DT-MC-OS` または `DT-MC-OX`：移管・メール継続・旧サイトあり

**最高難度。MX/SPF設定ミスでメール停止のリスクあり。**

**標準フロー：**
1. 既存メールサーバの**MX/SPF/DKIM/DMARC値を全て取得**
2. AuthCode取得・移管申請
3. AWS Route 53に**メール専用Aレコード**を作成（`mail.example.jp`）
4. MXレコードを既存メールサーバへ向ける設定
5. studioとの紐付け（WebはstudioへAレコード）
6. メール疎通テスト
7. 公開日当日：studio公開・リダイレクト実施

**特に注意：**
- DNS切替日とメール停止許容時間を必ず園と合意
- メールサーバのIPアドレスも確認（メール専用Aレコードに使用）
- 公開前に必ずメール送受信テスト

**所要期間目安：** 4〜6週間

## リスク判定ロジック

リスク判定は以下の条件で自動算出されます。

```javascript
function calcRisk(state) {
  // High: 既存メール継続 = 設定ミスでメール停止リスク
  if (state.mail === 'continue') return 'high';
  
  // Mid: 移管あり または リダイレクト他社依頼
  if (state.domain === 'transfer' || state.redirect === 'external') return 'mid';
  
  // Low: それ以外
  return 'low';
}
```

### 各リスクレベルの対応方針

| リスク | テクサポ確認 | リードタイム | メール停止リスク |
| :--- | :--- | :--- | :--- |
| **高** | 必須・詳細 | 4〜6週間推奨 | あり |
| **中** | 必須 | 3〜4週間推奨 | なし |
| **低** | 通常 | 2〜3週間推奨 | なし |

## パターンが該当しない場合

ヒアリング結果が想定パターンに該当しない場合、または特殊な対応が必要な場合は以下に該当する可能性があります。

- 複数ドメインの並行運用
- サブドメイン運用
- レンタルサーバへのAレコード設定
- DKIM/DMARCの厳格な維持が必要

これらのケースは個別判断が必要なため、テクサポへ相談してください。
